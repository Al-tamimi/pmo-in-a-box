# 02 · Design

**Goal: define what the PMO is, what it does, and the value it will deliver.**

This is where the PMO takes shape on paper: its mandate, the services it offers, and how
it will prove its worth.

| File | What it's for |
|---|---|
| `PMO-Charter.docx` | The founding document: mandate, authority, scope, objectives, and governance. Get this approved by leadership. |
| `PMO-Operating-Model.docx` | How the PMO operates and what *type* it is (supportive, controlling, or directive): structure, roles, and ways of working. |
| `PMO-Service-Catalogue.docx` | The menu of services the PMO offers its stakeholders, so expectations are explicit. |
| `Benefits-Realization-Approach.docx` | How benefits will be identified, planned, and tracked through to realization. |
| `Benefits-Realization-Plan-Template.docx` | A reusable plan template to track benefits for a specific initiative. |
| `Case-Study-Template.docx` | Capture and communicate success stories to build credibility. |

### How to use this stage
1. Draft the **Charter** first, nothing else is stable until the mandate is agreed.
2. Define the **Operating Model** and publish a **Service Catalogue** so stakeholders
   know what to expect.
3. Stand up the **Benefits Realization** approach early, it's what turns a PMO from a
   cost center into a value center.

**Next:** [`03-build`](../03-build), put the operational machinery in place.
