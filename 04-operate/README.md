# 04 · Operate

**Goal: run projects consistently with one shared, proven template set.**

These 11 templates cover the project life cycle from initiation to closure. Used together,
they give every project the same backbone, which is exactly what makes a portfolio
reportable and comparable.

| # | File | What it's for |
|:--:|---|---|
| 01 | `01-Project-Charter.docx` | Authorize the project and define its high-level intent. |
| 02 | `02-Business-Case.docx` | Justify the investment: problem, options, costs, benefits. *(Includes a de-identified worked example.)* |
| 03 | `03-Business-System-Requirements.docx` | Capture business and system requirements. |
| 04 | `04-Scope-of-Work.docx` | Define what's in and out of scope. |
| 05 | `05-RACI-Matrix.docx` | Make accountability explicit (Responsible / Accountable / Consulted / Informed). |
| 06 | `06-Stakeholder-Register-and-Communication-Plan.docx` | Identify stakeholders and plan how you'll communicate. |
| 07 | `07-Issue-Log.docx` | Track and resolve issues. |
| 08 | `08-Risk-Log.docx` | Identify, assess, and manage risks. |
| 09 | `09-Change-Request-Log.docx` | Control changes to scope, time, and cost. |
| 10 | `10-Lessons-Learned-and-Closure-Log.docx` | Close out cleanly and capture lessons. |
| 11 | `11-Project-Classification.docx` | Size/classify projects so governance is proportionate. |

### How to use this stage
1. Start every project with the **Charter** and **Business Case**.
2. Keep the **Risk**, **Issue**, and **Change** logs live, they're the heartbeat of
   control.
3. Always finish with **Lessons Learned**, it's how the PMO gets smarter over time.
