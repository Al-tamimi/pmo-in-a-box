# 05 · Report & Improve

**Goal: make the PMO's value visible, and keep getting better.**

A PMO that can't show its impact won't survive its first budget review. This stage turns
delivery data into clear signals for decision-makers, and closes the loop back to
maturity.

| File | What it's for |
|---|---|
| `PMO-KPI-Dashboard.xlsx` | A KPI dashboard to track portfolio health, delivery, and PMO performance at a glance. |
| `Status-Report-Template.docx` | A clean, repeatable status report for projects and the portfolio. |

### How to use this stage
1. Agree a **small set of KPIs** that matter to your leadership, don't measure
   everything.
2. Publish the **Status Report** on a fixed cadence (the rhythm matters more than the
   format).
3. Feed what you learn back into [`01-assess`](../01-assess) and re-baseline your P3M3
   maturity periodically. Improvement is a loop, not a line.

**Back to the start:** [`00-start-here`](../00-start-here) · re-assess and raise the bar.
