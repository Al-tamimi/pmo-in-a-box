# 03 · Build

**Goal: put the operational machinery in place so the PMO can actually run.**

Design becomes reality here: governance, methods, plans, and people.

| File | What it's for |
|---|---|
| `Governance-Plan.docx` | Decision rights, boards, gates, and escalation paths: how decisions get made and overseen. |
| `Management-and-Operational-Plan.docx` | How the PMO runs day to day: processes, cadences, and responsibilities. |
| `PMO-Methodologies.docx` | The delivery methodologies the PMO will standardize on (e.g., waterfall, agile, hybrid). |
| `Deliverables-Management-Plan.docx` | How deliverables are defined, produced, reviewed, and accepted. |
| `Audit-Template.docx` | Assure that projects follow the agreed standards: a repeatable project audit. |
| `Recruitment-Plan.docx` | Plan the roles and capabilities you need to staff the PMO. |

### How to use this stage
1. Lock down **Governance** first: boards, gates, and decision rights.
2. Document **how the PMO operates** and the **methodologies** projects must follow.
3. Use the **Audit** template to keep standards real, and the **Recruitment Plan** to
   build the team.

**Next:** [`04-operate`](../04-operate), run projects with a consistent template set.
